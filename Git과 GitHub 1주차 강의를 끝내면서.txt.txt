1. 앞으로 git 프로젝트로 만들고 싶은 것:
Git과 Github를 통하여 어떠한 파일이 생성되었고, 나아가 생성된 파일이 어떻게 수정되어있는지 History를 통하여 볼 수 있게 되었다.
내 컴퓨터에만 저장되어있는 Local Repo를 Remote Repo로 Tracking (연결)하여 팀원들과 앞으로 있을 항해99 프로젝트를 성실히 임하고 싶다.
1주차에는 배정받은 팀원님들과 어떠한 과제를 하게 될지 궁금하다.

2. 1주차 주요 개념 키워드 적어보기:
이미 Toyproject를 통하여 Git/Github을 써보았던 경험이 있었지만, 1주차 강의를 통하여 새롭게 알게된 키워드를 하기로 정리해본다.
a) Tracking(추적): 로컬 repo와 원격 repo를 연결한다!
b) Clone: 원격 repo를 내 컴퓨터에 가져와서 초기 repo 세팅하는 것을 일컫는다.
c) 버전 관리: 프로젝트 상태가 변경되는 정보를 알고 있다는 것입니다. Git은 가장 널리 쓰이는 버전관리 도구 중에 하나로 commit 을 사용해서 버전이 달라지는 것을 관리합니다.
d) Commit id: Commit을 구분하기 위해 Git이 붙여주는 아이디라고 생각하면 된다.
